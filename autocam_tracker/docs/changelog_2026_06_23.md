# AutoCam Tracker - 系統改版與優化紀錄 (2026-06-23)

本次更新著重於大幅簡化使用者的操作流程、提升高密度車流下的運算效能，並修復了多項導致系統癱瘓或判定失效的深層邏輯 Bug。

## 1. UX/UI 介面與操作邏輯大翻新
* **移除繁瑣的點擊模式切換**：廢除了原本需要手動切換 Radio Button (Normal Select / Create New GID) 的舊設計。
* **導入直覺的鍵盤快捷鍵**：
  * `Shift + 左鍵`：直接建立全新的目標 (New GID) 並擷取特徵。
  * `Ctrl + 左鍵`：為目前選定的目標補充特徵 (Add Feature)。
* **新增右鍵快捷選單**：在畫面上對車輛點擊右鍵，可直接喚出選單，將該車輛指派給現有 GID 或建立新 GID。
* **新增操作指南按鈕**：於介面左上方新增「Help / Controls」按鈕，隨時可查看熱鍵提示。
* **即時特徵數量回饋**：左下角的資料庫 (Anchor DB) 列表現在會即時顯示每個 GID 與 Shot 累積的特徵數量（例如：`car G1 (3 Feat)`），讓操作者能明確知道特徵已成功寫入。

## 2. 核心效能最佳化 (解決車多卡頓與 OOM 崩潰)
* **降低推論解析度與模型升級**：將 YOLO 推論預設解析度由 `640` 降至 `480`，並將預設模型升級為 `yolo11n`，大幅提升基礎幀率。
* **ReID 雷達範圍限制 (Limited ReID Scope)**：
  * 過去系統每 15 幀自動糾錯時，會強制對畫面上「所有車輛」進行特徵擷取，導致嚴重卡頓。
  * **優化後**：系統只會對「目前追蹤目標周圍 300 像素半徑內」的車輛（通常為 2~4 台）進行特徵比對，將運算時間從數十毫秒壓縮至幾毫秒。
* **精準單體特徵擷取 (防 OOM 保護)**：
  * 過去在使用快捷鍵綁定特徵時，系統也會連帶對全畫面車輛抽特徵，若畫面超過 50 台車，會直接導致顯示卡記憶體耗盡 (OOM)，並觸發底層的永久停用保護機制，導致後續點擊完全失效。
  * **優化後**：按下快捷鍵綁定時，系統「只會」針對滑鼠點擊的那一台車進行特徵擷取，徹底消滅 OOM 崩潰問題。

## 3. 跨機位追蹤邏輯強化 (Camera Cut)
* **修復跨機位 ReID 失效 Bug**：舊版系統在切換機位 (Shot) 後，因為新機位尚無特徵資料，會直接放棄比對。現已修改底層資料庫，當新機位無資料時，會智能地回頭翻找「所有歷史機位」的特徵來進行比對。
* **跨機位自動學習機制**：當系統透過歷史特徵在新機位成功認出車輛後，會**全自動擷取該車輛在新機位的樣貌**並寫入資料庫中。使用者無需手動操作，下方的 TreeView 就會自動長出 `Shot 2`、`Shot 3` 等節點。

## 4. 基礎設施與互動細節
* **暫停模式下的單步執行 (Step-One-Frame)**：過去在暫停影片時點擊綁定特徵，系統雖然有記錄，但不會立刻更新 UI，讓使用者誤以為功能壞掉。現在暫停時的任何綁定操作，都會瞬間喚醒系統「精準前進 1 幀」並立刻完成資料寫入與 UI 更新。
* **加入 Reset DB 功能**：左側控制面板新增 `Reset DB` 按鈕，可一鍵清除所有已記錄的特徵與追蹤狀態。
* **A/V 同步保護**：在 Pipeline 引入自動丟幀 (Drop Frame) 邏輯，確保在極端運算負載下，影像播放速度仍能盡量與現實時間（或音軌）同步。
